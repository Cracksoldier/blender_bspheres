bSpheres NX

zSpheres-style base-mesh creation for Blender.

Blender 4.2 LTS → 5.x
Download latest release View on GitHub Report an Issue

What it is

zBrush has a handy tool called zSpheres for blocking out quick base meshes. Blender can do the same thing with a single-vertex mesh and the Mirror, Skin, and Subdivision Surface modifiers — but the setup is fiddly. bSpheres NX wraps all of that behind one panel so you can just start sketching, then bake the result into a watertight, sculptable mesh.

Install — with automatic updates

  1. In Blender, open Edit → Preferences → Get Extensions, expand the Repositories dropdown (top right) and choose + → Add Remote Repository.
  2. Enter https://cracksoldier.github.io/blender_bspheres/index.json and enable Check for Updates on Startup.
  3. Search for bSpheres NX in Get Extensions and click Install. New releases now arrive as in-place updates — no reinstall needed.
  4. Press N in the 3D Viewport and open the bSpheres NX tab in the sidebar.

Prefer a manual install? Download bspheres-<version>.zip from the Releases page and use the Repositories dropdown → Install from Disk… instead. If you previously installed from disk and want automatic updates, uninstall that copy once and install from the repository — per-object settings in your .blend files are unaffected.

Typical workflow

  1. Press N in the 3D Viewport, open the bSpheres NX tab, and click Create. This builds a single-vertex mesh with Mirror, Skin, and Subdivision modifiers already stacked.
  2. Enter Edit Mode (Tab). Use E to extrude new joints, Ctrl + A to scale the skin at the selected vertex (native Blender Skin modifier shortcut), Ctrl + R to select two vertices then click between them to add a midpoint, and G to reposition joints. Toggle Mirror axes in the panel for symmetry while you work.
  3. Click Preview / Refresh at any point to generate a temporary baked mesh in bSpheres_Preview without leaving the control object. Click it again after any edit to update in-place.
  4. Tune bSkin Settings — voxel remesh size, smooth shading, merge doubles — until the previewed output looks right. Or pick an Output Preset (Organic Smooth, Tentacles, 3D Print Solid, …) and click Apply to set everything at once.
  5. When satisfied, click Make bSkin to produce a permanent bake in bSpheres_Output (non-destructive — the control mesh is untouched and you can bake again any time), or Make Rigged bSkin to also generate the armature and bind it with automatic weights. Or click Apply to bake directly onto the control object when you are fully done iterating.
  6. Armature workflow (optional): If you want a different vertex to be the skeleton root (the initial vertex is auto-marked on Create), select it in Edit Mode and use Mark Root in the panel. Then click Generate Armature from any mode — the operator switches to Object Mode automatically. Each edge becomes one bone, parented root-outward, with mirrored halves included. The result lands in bSpheres_Armatures.

Quick guide — the panel

Create

Builds the single-vertex mesh with the Mirror, Skin, and Subdivision modifiers already set up.

Mirror Axis X / Y / Z

Toggle symmetry on each axis while you work.

Mark Root

Set the selected vertex as the skin root — handy when building bones from the vertices.

Mark / Clear Loose

Make vertices act like a grab brush, pulling the skin toward them instead of centering it.

Viewport

How much to subdivide the skin. Higher = smoother preview.

bSkin Settings

Per-object output controls: voxel remesh toggle + size, shade-smooth, merge doubles + threshold, recalculate normals, and thin-branch warning + min radius. Applied by Make bSkin, Preview, and Apply.

Output Presets

Seven curated setting combos — Organic Smooth, Humanoid Basemesh, Creature Limbs, Tentacles, Hard Mannequin, Low Poly Blockout, and 3D Print Solid. Apply sets all bSkin Settings plus the Subdivision level in one click; individual settings can still be tweaked afterwards.

Selected Node

Edit Mode only. Shows the active vertex's live skin radius and root/loose flags. Set Radius opens a dialog to type exact X/Y radii for all selected vertices. Mark / Clear Preserve adds or removes the vertex from the thin-branch warning exemption. Select Children / Select Parents selects all downstream or upstream vertices from the active joint — useful for posing a full limb as a unit.

Insert Meshes

Edit Mode only. Assign any mesh object to the active vertex (Node) or to the edge toward its parent (Link); Clear Active Node removes both. Refresh Insert Meshes then places instances in a bSpheres_Inserts collection — node meshes at their vertex, link meshes aligned and stretched along their edge. Re-run Refresh after topology edits.

Branch Tools

Edit Mode only. Duplicate Branch copies the active vertex and everything downstream, leaving the copy selected for moving. Mirror X / Y / Z mirrors the downstream branch across the chosen object-local axis, and Radial Duplicate makes rotated copies around the axis — both automatically connect the new branches back to the active vertex. Taper Branch interpolates skin radii from the active joint down to an end radius at the tips.

Generate Armature

Converts the bSphere control mesh into a Blender armature: each edge becomes one bone, parented to follow the vertex graph from the skin root. Bones for the mirrored halves are generated too, matching the Mirror modifier's axes (untick Include Mirrored Half in the redo panel to skip them). Output goes into a bSpheres_Armatures collection.

Preview / Refresh

On-demand preview in a bSpheres_Preview collection. Re-clicking updates the same object in-place. Use Delete to remove it.

Make bSkin

Non-destructive permanent bake. Creates a new mesh in bSpheres_Output without touching the control structure — iterate as many times as you like.

Make Rigged bSkin

The full pipeline in one click: bakes a bSkin, generates the full-skeleton armature, and binds the mesh to it with automatic weights. The result is ready to pose.

Apply

Destructive bake. Applies the three modifiers onto the control object, then post-processes using the same bSkin Settings. Use when done iterating.

bSkin Settings reference

These settings are stored per control object and applied by Make bSkin, Preview / Refresh, and Apply. The Preset dropdown at the top of the section applies a curated combination of them (plus a Subdivision level) in one click.

Setting Default Range Effect
Remesh✓—Runs Blender's voxel remesher on the baked skin to produce a clean, uniform mesh.
Voxel Size0.020.001 – 1.0Grid cell size for remeshing. Smaller = more polygons and finer detail; larger = coarser, lighter mesh.
Shade Smooth✓—Sets shade-smooth on the output so it renders without hard faceting.

Merge Distance0.0010.00001 – 0.1Distance used by merge-by-distance. Only active when Merge Doubles is enabled.
Recalculate Normals✗—Recalculates face normals to point outward. Helpful if some faces appear inverted after baking.

Known limitations

Armature mirroring is positional

Mirrored bone sets are generated by mirroring bone positions across the Mirror modifier's axes in the control object's local space. Mirror offsets or a mirror object on the modifier are not taken into account — keep the Mirror modifier at its default settings for a matching skeleton.

Armature requires a skin root

Generate Armature cancels with an error if no vertex has the Skin Root flag set. Mark one vertex as root in Edit Mode using Mark Root in the panel (or Blender's built-in Skin → Mark Root shortcut) before generating.

Cyclic edges are skipped

If the control mesh contains a loop (an edge that closes a ring), the back-edge is skipped during BFS traversal and reported as a warning. The armature is still generated, but those edges produce no bones.

Disconnected components excluded

Vertices that are not connected to the skin root (directly or via a chain of edges) are excluded from armature generation. A warning lists the count. Make sure every branch traces back to the root vertex.

Zero-length edges produce no bones

Edges shorter than 1×10⁻⁶ units in world space are skipped. Dissolve or merge vertices that share the same position to avoid gaps in the generated skeleton.

Voxel remesh changes topology

When Voxel Remesh is enabled, the output is rebuilt as a uniform voxel grid. The original skin vertex/edge layout is not preserved, which may be unexpected if you planned to use the mesh as a retopology reference.

Operators act on the active object

Make bSkin, Preview, and Apply run on whichever bSphere control mesh is currently active. When multiple control meshes are in the scene, confirm the right object is active before baking.

Chain selection requires a skin root

Select Children and Select Parents both use BFS from the skin root. If no root is marked, or the active vertex is disconnected from the root, the operator cancels with a warning in the status bar. The same applies to the Branch Tools and Link Mesh assignment.

Insert meshes are visual by default

Insert-mesh instances live in the bSpheres_Inserts collection and are not merged into baked output unless Include Inserts is enabled in bSkin Settings — with it on, all bake paths join them before remeshing.

Insert instances track indices

Refresh Insert Meshes matches instances to vertices and edges by index. After adding or deleting vertices, run Refresh Insert Meshes again so instances follow the new layout — assignments on deleted elements are cleaned up, but shifted indices can briefly point at the wrong joint until refreshed. Manual transform tweaks on instances are stored relative to their joint, so they survive refreshes and follow the joint in bakes.
