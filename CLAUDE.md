# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A Blender addon (`bSpheres NX`) that emulates zBrush's zSpheres for fast base-mesh creation. It wraps Blender's existing single-vertex + Mirror/Skin/Subdivision-Surface modifier workflow behind a single panel so users can sketch base meshes by extruding/scaling vertices, then bake the result into a sculptable mesh.

This is a fork of [PapaTemporal/blender_bspheres](https://github.com/PapaTemporal/blender_bspheres), updated for the Blender Extensions system (4.2+).

Target runtime: **Blender 4.2 LTS through 5.x**, packaged as an **Extension**. Metadata lives in `blender_manifest.toml` (`blender_version_min = "4.2.0"`), not in a `bl_info` dict — the legacy `bl_info` blocks were removed when the addon was converted to the Extensions system. The code is `bpy`/`bmesh` only — there is no lint or test tooling, no third-party dependency, and no entry point outside Blender.

## Running and testing

There is no CLI or test harness — the addon only runs inside Blender. It is an Extension, so install/build go through the extension tooling:

- **Validate manifest:** `blender --command extension validate .` (from the repo folder).
- **Build:** `blender --command extension build` → produces `bspheres-<version>.zip`.
- **Install:** Blender *Preferences → Get Extensions → Install from Disk*, select the zip, enable it. The panel appears in the 3D Viewport N-panel under the **bSpheres NX** tab.
- **Iterate:** edit the `.py` files and toggle the extension off/on (or use *Reload Scripts*). Because it loads as a package, the relative import `from . bSpheres import *` in `__init__.py` resolves normally.

Validate changes by exercising the full flow manually in Blender: **Create** → extrude/scale/move verts → adjust Mirror axes, Subdivision level, and bSkin Settings → **Preview / Refresh** → **Make bSkin** → **Apply**. There is no automated coverage to rely on.

## CI / release

- **`.github/workflows/ci.yml`** — runs on every branch push and PR: byte-compiles the Python files and runs `blender --command extension validate`. Tag pushes are skipped (handled by the release workflow).
- **`.github/workflows/release.yml`** — manual `workflow_dispatch`. Takes a version string, bumps `blender_manifest.toml`, downloads Blender 4.2 LTS into `/tmp` (not the repo root), validates and builds the extension zip, commits the bump, pushes a `v<version>` tag, and publishes a GitHub Release with the zip attached.

## Docs

- **`docs/index.html`** — self-contained GitHub Pages landing page (no external dependencies). Deployed via Pages with source = `main /docs`.
- **`docs/favicon.svg`** — orange sphere icon matching the site's accent colour.
- Both files are excluded from the packaged extension via `[build].paths_exclude_pattern` in the manifest.

## Architecture

Two Python files plus the extension manifest:

- **`blender_manifest.toml`** — extension metadata (id, version, `blender_version_min`, license, tags). This is the source of truth for what used to be `bl_info`.
- **`__init__.py`** — entry point: `register()`/`unregister()` register all classes from `bSpheres.py`. `BSpheresPreferences` is registered first; `BSpheresSkinSettings` second (before the `PointerProperty` assignment); `BSpheresPreferences` unregistered last. No `bl_info` (the manifest replaces it).
- **`bSpheres.py`** — all behavior: one `AddonPreferences`, one `PropertyGroup`, the operators, one panel, and a set of module-level helper functions.

The registered classes and how they fit together:

- **`BSpheresPreferences`** (`bpy.types.AddonPreferences`) — addon-wide settings exposed in **Edit → Preferences → Add-ons → bSpheres NX**. Currently holds one property: `default_mirror_axes` (`BoolVectorProperty`, size=3, default `(False, True, False)` = Y only). Read by `AddBMesh` via `bpy.context.preferences.addons.get(__package__)` with a `(False, True, False)` fallback if the key is absent. `bl_idname = __package__` resolves to the full extension package path (e.g. `bl_ext.user_default.bspheres`), which is the key Blender uses in `bpy.context.preferences.addons`.
- **`BSpheresSkinSettings`** (`bpy.types.PropertyGroup`) — per-object output settings: `voxel_size`, `use_voxel_remesh`, `use_smooth_shading`, `use_merge_doubles`, `merge_threshold`, `use_recalc_normals`, `use_include_inserts`, `warn_thin_branches`, `min_branch_radius`, plus UI state for the insert-mesh pickers (`insert_node_mesh_name`, `insert_link_mesh_name`) and the preset dropdown (`last_preset`). Attached to every Blender object via `bpy.types.Object.bspheres_skin_settings`. Persists in the `.blend` file. Read by `MakeBSkin`, `MakeRiggedBSkin`, `PreviewBSkin`, and `applyBSphereModifiers` to control post-bake processing.
- **`AddBMesh`** (`mesh.primitive_bsphere_add`, "Create" button) — builds an 8-vertex box via `add_box()`, merges all verts to center to get a single vertex, then stacks three modifiers (**Mirror**, **Skin**, **Subdivision**), enters Edit mode with x-ray on, and marks the skin root. The Mirror modifier's initial `use_axis` is read from `BSpheresPreferences.default_mirror_axes` (falls back to `(False, True, False)` if prefs are unavailable). This single-vertex-plus-skin setup *is* the "zSphere" the user then extrudes.
- **`MakeBSkin`** (`bspheres.make_bskin`, "Make bSkin" button) — non-destructive permanent bake. Delegates the bake to `_bake_bskin_object`, optionally joins insert meshes (`use_include_inserts`), applies post-bake settings (remesh, smooth), and returns the user to their previous mode. The control object is untouched; each run produces a new output object.
- **`MakeRiggedBSkin`** (`bspheres.make_rigged_bskin`, "Make Rigged bSkin" button) — composes `_bake_bskin_object` + `_apply_bskin_settings` + `_generate_armature_object(include_mirrored=True)`, then parents the baked mesh to the armature with automatic weights (`parent_set(type='ARMATURE_AUTO')` inside a selection save/restore `try/finally`). If no skin root is marked the bake stays but rigging is cancelled with an error.
- **`PreviewBSkin`** (`bspheres.preview_bskin`, "Preview / Refresh" button) — non-destructive on-demand preview. Same bake logic as `MakeBSkin` but places output in `bSpheres_Preview` and tags it with `obj["bspheres_preview"] = True` / `obj["bspheres_source"] = source_name`. On re-run, replaces the existing preview object's mesh data in-place (same object, updated geometry) so the Outliner stays clean.
- **`DeleteBSkinPreview`** (`bspheres.delete_bskin_preview`, "Delete" button) — removes the preview object and its mesh datablock for the active bSphere. Poll is wired to `_find_preview_obj` so the button greys out automatically when no preview exists.
- **`applyBSphereModifiers`** (`tcg.apply_bsphere_modifiers`, "Apply" button) — **destructive** bake. Applies the three modifiers directly onto the control object, then conditionally voxel-remeshes, runs `_run_mesh_cleanup`, and shade-smooths using the same `bspheres_skin_settings` as the non-destructive operators. Use when done iterating.
- **`BSphereMarkPreserve`** (`bspheres.mark_preserve`, "Mark Preserve" button) — adds selected vertices in EDIT mode to the `bspheres_preserve` vertex group. Preserved vertices are skipped by `_warn_thin_branches`. Poll-guarded to EDIT_MESH mode on bSphere control objects only. Saves/restores `vertex_groups.active_index` so the user's active group is not clobbered.
- **`BSphereClearPreserve`** (`bspheres.clear_preserve`, "Clear Preserve" button) — removes selected vertices from the `bspheres_preserve` group. Same guard and save/restore pattern as `BSphereMarkPreserve`.
- **`GenerateBSphereArmature`** (`bspheres.generate_armature`, "Generate Armature" button) — thin wrapper around `_generate_armature_object`: captures the mode, switches to OBJECT, calls the helper, restores the mode. Its `include_mirrored` operator property (default True, in the redo panel) controls whether mirrored bone sets are generated; when off, only the unmirrored half is produced. Poll uses `_is_bsphere_control`.
- **`BSphereSelectChildChain`** (`bspheres.select_child_chain`, "Select Children" button) — EDIT_MESH only. Delegates all BMesh setup, root/active-vertex validation, and BFS to `_get_chain_graph`, then inverts `parent_map` into a `children_map` and BFS-traverses downstream from the active vertex to select all descendant vertices.
- **`BSphereSelectParentChain`** (`bspheres.select_parent_chain`, "Select Parents" button) — EDIT_MESH only. Delegates setup to `_get_chain_graph`, then walks `parent_map` from the active vertex up to the root, selecting all ancestor vertices (exclusive of active, inclusive of root).
- **Branch/radius tools** (all EDIT_MESH, poll `_is_bsphere_control`): `BSpheresDuplicateBranch` / `BSpheresMirrorBranch` / `BSpheresRadialDuplicate` duplicate the downstream branch via `_get_branch_geom` (mirror/radial exclude the active vertex and re-attach the copies to it with new edges). `BSphereRotateBranch` rotates the active vertex plus everything downstream around its **parent** joint (`parent_map` pivot; CANCELLED with INFO when the active vertex is the root) — zSphere-style limb posing via the redo panel. `BSphereTaperBranch` lerps skin radii from the active vertex to an end radius by geometric path distance. `BSphereRelaxRadii` neighbor-averages the skin radii of selected vertices over N Jacobi iterations (snapshot per pass); `bspheres_preserve` vertices contribute as neighbors but are never modified; needs no root/active, so it reads the BMesh directly instead of `_get_chain_graph`. `BSphereSetSkinRadius` opens a dialog pre-filled from the active vertex and writes exact X/Y radii to all selected vertices.
- **`BSpheresPanel`** (`OBJECT_PT_bSpheres_Panel`) — the UI. Introspects `context.object.modifiers` and surfaces the live Mirror axes, Subdivision `levels`, and Skin loose/root marking. The "bSkin Settings", "Preview", "Armature", and "Convert" sections are only drawn when `_is_bsphere_control(obj)` is true. In EDIT_MESH mode, also draws a "Selected Node" section when a vertex is active in `bm.select_history`, showing live skin radius and root/loose flags (read from `bm.verts.layers.skin`, not from the RNA mesh layer) plus Mark/Clear Preserve buttons and a "Chain Selection" row (Select Children / Select Parents). The "Branch Tools" box is drawn for any EDIT_MESH session on a control object — deliberately *not* gated on an active vertex, because Relax Radii only needs a selection (box-select leaves `select_history` empty) and the active-vertex tools report a warning when none is set.

### Module-level helpers in `bSpheres.py`

- **`_is_bsphere_control(obj)`** — returns True if `obj` is a mesh with MIRROR, SKIN, and SUBSURF modifiers all present. Used as the `poll` guard for `MakeBSkin`, `PreviewBSkin`, `applyBSphereModifiers`, and the settings/preview panel section.
- **`_ensure_collection(name, scene)`** — finds or creates a named collection and ensures it is linked to the current scene. Used by `MakeBSkin` (`bSpheres_Output`), `PreviewBSkin` (`bSpheres_Preview`), and `GenerateBSphereArmature` (`bSpheres_Armatures`).
- **`_apply_bskin_settings(obj, settings, context)`** — applies post-bake processing to a mesh object: voxel remesh (with active+selected object swap), then `_run_mesh_cleanup`, then shade smooth. Order matters: cleanup and smooth run after remesh so they apply to the final geometry.
- **`_find_preview_obj(source_name)`** — searches `bSpheres_Preview` for an object tagged with `bspheres_source == source_name`. Used by `PreviewBSkin` (refresh path) and `DeleteBSkinPreview.poll`.
- **`_warn_thin_branches(operator, source_obj, settings)`** — iterates the source object's skin vertices (read in OBJECT mode after `mode_set` flush) and calls `operator.report({'WARNING'}, …)` for any vertex whose skin radius is below `settings.min_branch_radius`. Skips vertices that are in the `bspheres_preserve` vertex group. Must be called **after** `mode_set(OBJECT)` so the RNA mesh layer reflects the current BMesh state.
- **`_run_mesh_cleanup(obj, settings, context)`** — enters EDIT mode on `obj`, optionally runs `mesh.merge_by_distance` and/or `mesh.normals_make_consistent`, then returns to OBJECT mode. Deselects all other selected objects first so `mode_set(EDIT)` cannot enter multi-object edit and run the cleanup on them (e.g. the still-selected control object during a non-destructive bake). Saves/restores the active object and all selection state. Uses `try/finally` so everything is restored even if an operator raises. No-ops immediately if both `use_merge_doubles` and `use_recalc_normals` are False.
- **`_build_mesh_graph(obj)`** — builds an adjacency dict `{vertex_index: [neighbor_index, …]}` from `obj.data.edges`. Must be called in OBJECT mode (RNA `obj.data.edges` is stale in EDIT mode until the BMesh is flushed). Used by `GenerateBSphereArmature`.
- **`_find_skin_root(obj)`** — scans `obj.data.skin_vertices[0].data` and returns the index of the first vertex with `use_root == True`, or `-1` if none. Must be called in OBJECT mode for the same reason as `_build_mesh_graph`. Used by `GenerateBSphereArmature`.
- **`_bfs_tree(adj, root_idx)`** — BFS over the adjacency dict from `root_idx`. Returns `(parent_map, found_cycle)`: `parent_map` maps each reached vertex index to its BFS parent (root maps to `None`) — callers use `len(parent_map)` and `key in parent_map` instead of a separate visited set; `found_cycle` is `True` if any back-edge was detected. Pure Python — no Blender calls. Used by `GenerateBSphereArmature` and `_get_chain_graph`.
- **`_bfs_from_root(bm)`** — quiet BFS over the live BMesh from the skin root. Calls `bm.verts.index_update()` / `bm.edges.index_update()` first (BMesh element indices go stale — new elements report `-1` — after topology edits in the same edit session), then `ensure_lookup_table()`, locates the root via `bm.verts.layers.skin[0]`, builds adjacency from `v.link_edges`, and runs `_bfs_tree`. Returns `(parent_map, found_cycle, error)` where `error` is a warning string (and `parent_map` is `None`) if there is no skin layer or no root. Every index-keyed BMesh path goes through this, so indices are always fresh.
- **`_get_chain_graph(operator, context)`** — shared EDIT-mode setup for the chain-selection, branch, and link-mesh operators. Opens the BMesh, delegates the graph work to `_bfs_from_root`, validates the active element is a `BMVert`, checks reachability and cycles, and emits warnings via `operator.report`. Returns `(bm, parent_map, active_vert)` on success or `None` if any precondition fails. Safe to call in EDIT mode — reads the live BMesh, not the RNA layer.
- **`_find_parent_edge(bm, parent_map, active_vert)`** — returns the `BMEdge` connecting `active_vert` to its BFS parent, or `None` if the vertex is the root. `_find_parent_edge_idx(bm, active_vert)` wraps it (running `_bfs_from_root` itself, so it also refreshes indices) and returns the edge index or `-1`.
- **`_build_children_map(parent_map)`** — inverts a BFS parent_map into `{parent_index: [child_index, …]}`. `_direct_child_verts(bm, parent_map, parent_idx)` resolves the BMVerts whose BFS parent is `parent_idx` (lookup table must be fresh). Used by the chain-selection, branch, and taper operators.
- **`_bake_bskin_object(context, source_obj)`** — bakes the evaluated modifier stack into a new mesh object in `bSpheres_Output` (naming: `bSphere…` → `bSkin…`), copying `matrix_world`. Must be called in OBJECT mode. Used by `MakeBSkin` and `MakeRiggedBSkin`.
- **`_generate_armature_object(operator, context, obj, include_mirrored=True)`** — creates the armature in `bSpheres_Armatures`: skin root via `_find_skin_root`, BFS via `_bfs_tree`, one bone per edge, parented per the traversal tree. With `include_mirrored`, adds one mirrored bone set per non-empty subset of the Mirror modifier's enabled axes, mirrored in the control object's **local** space (`armature.symmetrize` was deliberately avoided — it only handles X, while bSpheres defaults to Y). Edges lying entirely on the mirror plane are skipped as duplicates; a mirrored chain whose parent edge was skipped parents to the original on-plane bone. Warns on cycles, unreachable vertices, zero-length edges, and orphaned bones. Must be called in OBJECT mode; the EDIT_ARMATURE session is wrapped in `try/finally` restoring selection/active and leaving the mode in OBJECT. Returns the armature object, or `None` if no root is marked.
- **`_iter_insert_placements(operator, obj)`** — yields `(kind, index, source_obj, matrix_world)` for every valid insert-mesh assignment (`'VERT'` keeps the source's world rotation/scale with the translation replaced by the vertex position; `'EDGE'` aligns local +Z along the edge and stretches to its length). Owns the missing/non-mesh warnings and the zero-length-edge skip. Must be called in OBJECT mode. Consumed by `BSphereRefreshInsertMeshes` (computed transforms) and `_collect_insert_placements`.
- **`_find_insert_instances(source_name)`** — returns `({vert_idx: inst}, {edge_idx: inst})` for the live instances in `bSpheres_Inserts` tagged with the given source name. Used by `BSphereRefreshInsertMeshes` and `_collect_insert_placements`.
- **`_collect_insert_placements(operator, source_obj)`** — bake-path placement list: computed transforms from `_iter_insert_placements`, resolved per instance via `_resolve_instance_matrix` (untweaked instances follow the current geometry; tweaked instances keep their delta). A stale instance (assignment changed without a refresh) keeps the computed transform and reports a WARNING; a legacy instance without a baseline wins with its raw `matrix_world`.
- **Instance-baseline helpers** — `_store_computed_matrix` / `_get_stored_computed_matrix` persist the computed placement on every instance as the `bspheres_computed_matrix` ID property (16 floats, written on every refresh). `_resolve_instance_matrix(inst, computed)` compares the instance's `matrix_world` against that baseline (`_matrices_close`, float32-tolerant) to separate deliberate user tweaks — re-applied as a local-frame delta `computed @ stored⁻¹ @ matrix_world` so they ride the joint — from mere staleness (snapped to the fresh placement); returns `None` for pre-baseline instances, and callers keep the legacy behavior once until the next refresh writes a baseline. `_apply_instance_matrix(inst, kind, matrix)` writes a placement to an instance (EDGE → quaternion channels to avoid gimbal-locked keyframing, VERT → `matrix_world`).
- **`_join_inserts(operator, context, target_obj, placements)`** — bakes each placement's source through the depsgraph (`new_from_object`, so insert modifiers apply), creates temp objects with the placement matrices, and `object.join`s them into `target_obj` (selection saved/restored via `try/finally`). No-ops on an empty list. Must be called in OBJECT mode. **`_maybe_join_inserts(operator, context, target_obj, source_obj)`** wraps it behind the `use_include_inserts` check and collects placements live — used by the three non-destructive bake paths; the destructive Apply path calls `_join_inserts` directly with pre-collected placements.

### Cross-file coupling to watch

- **Mode round-trip.** `AddBMesh.execute` stashes `context.mode` into `context.scene['previous_mode']`, and `applyBSphereModifiers.execute` reads it back. `MakeBSkin` and `PreviewBSkin` capture mode locally and restore it. All use `_MODE_SET_MAP` to normalize `EDIT_MESH` → `EDIT` etc. Remesh, cleanup, and smooth all run **before** `mode_set` restore — `voxel_remesh` and `_run_mesh_cleanup` both require OBJECT mode entry points. `_warn_thin_branches` must also be called **after** `mode_set(OBJECT)` (not before) so the RNA `skin_vertices` layer is flushed from the BMesh.
- **Settings coupling.** `BSpheresSkinSettings` is the single source of truth for all bake settings. All four bake operators (`MakeBSkin`, `MakeRiggedBSkin`, `PreviewBSkin`, `applyBSphereModifiers`) read from it. `applyBSphereModifiers` does NOT use the `_apply_bskin_settings` helper (the object is already active there, so no active-object swap is needed) but must be kept in sync with any logic changes. `use_include_inserts` joins insert meshes into all four bake outputs before remesh; the destructive Apply path must collect placements via `_collect_insert_placements` **before** applying modifiers, because modifier apply rebuilds the mesh and destroys the per-vertex/edge string attributes.
- **Per-node preserve flag.** The `bspheres_preserve` vertex group is the per-vertex flag for "preserve thickness". Vertex groups survive edit-mode extrude/delete more reliably than mesh attributes. `BSphereMarkPreserve` and `BSphereClearPreserve` manage this group; `_warn_thin_branches` reads it to skip preserved vertices.
- **Skin data in EDIT mode.** `obj.data.skin_vertices[0].data[i].radius` is the RNA mesh layer — only valid in OBJECT mode (stale in EDIT mode until `mode_set(OBJECT)` flushes the BMesh). Whenever skin vertex data must be read in EDIT mode (panel display), use `bm.verts.layers.skin[0]` on the already-open BMesh instead.
- **Modifier matching.** The panel and all operator polls locate modifiers by `modifier.type` (`MIRROR`/`SKIN`/`SUBSURF`), so `.001` name suffixes don't break them.
- **Collection hygiene.** `_ensure_collection` guards against multi-scene blend files by checking `context.scene.collection.children_recursive` before linking (recursive, so a collection the user nested elsewhere in the scene is not linked a second time at the root). `bSpheres_Output` (permanent bakes), `bSpheres_Preview` (temporary previews), and `bSpheres_Armatures` (generated armatures) are separate collections.
- The `width`/`height`/`depth` props on `AddBMesh` size the throwaway box before it is merged to a point, so they have no visible effect on the final single-vertex result.
- **String attributes are bytes in Blender 4.5+.** `StringAttributeValue.value` is `str` on 4.2 LTS but `bytes` on 4.5 and later. Every read/write of the `bspheres_node_mesh` / `bspheres_link_mesh` attribute values must go through `_get_string_attr` / `_set_string_attr`; a bare `.value` access works on 4.2 and silently breaks (or raises TypeError) on newer Blender.
- **Graph helpers — OBJECT mode vs BMesh.** `_build_mesh_graph` and `_find_skin_root` read the RNA mesh layer (`obj.data`) and must only be called after `mode_set(OBJECT)`. The chain-selection/branch/insert operators bypass this by going through `_bfs_from_root`, which reads the skin root directly from `bm.verts.layers.skin[0]` on the live BMesh and builds adjacency from `v.link_edges` — the same safe pattern used by the panel display. `_bfs_from_root` calls `index_update()` on verts and edges before any index-keyed use (stale/`-1` indices after topology edits would corrupt the adjacency dict or write mesh attributes to the wrong element); `BSphereAssignNodeMesh` does its own `bm.verts.index_update()` before reading `active.index` because it doesn't build the graph. The panel display needs neither because it only accesses `bm.select_history.active` by reference, not by index.
- **Armature generation — edit_bones window.** `arm_data.edit_bones` is only writable while the armature object is active and Blender is in EDIT mode. `_generate_armature_object` saves all currently-selected objects (`other_selected`), deselects them and the control mesh, switches active to `arm_obj`, then wraps the entire EDIT_ARMATURE session in `try/finally` so selection, active object, and mode are always restored — even if an exception propagates. The bone-creation, bone-parenting, and mirrored-copy passes all happen inside the same EDIT mode session; exiting between them would invalidate the edit-bone references needed for `bone.parent` assignment.
